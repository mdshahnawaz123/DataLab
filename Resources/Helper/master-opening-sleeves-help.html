Master Opening & Sleeves

Add-in Revit 2024 – 2026 Last updated: April 2025
The Master Opening & Sleeves tool automates the detection of clashes between MEP elements (pipes, ducts, cable trays) and structural/architectural host elements (walls, floors, ceilings, beams), and generates code-compliant openings or sleeve families at every intersection — eliminating the need for manual coordination.

1. MEP Elements

Select which MEP categories to include in the clash scan. The tool searches both the host model and all loaded Revit links.

Option Description Supported Families
Pipes Round and rectangular pipe runs including fittings and accessories. System families, in-place, loaded families
Ducts Rectangular, round, and oval duct segments and duct fittings. System families, in-place, loaded families
Cable Trays Ladder and solid-bottom cable trays and their fittings. System families, loaded families
ℹ
Note At least one MEP category must be selected before running Scan For Clashes. The status bar at the bottom of the dialog reports how many elements of each type are currently detected (e.g., "Model: 3 pipe(s), 2 duct(s), 0 cable tray(s)").

2. Host Elements

Host elements are the structural or architectural components that will be cut to create openings. Openings are always placed in the host model, even when the MEP element originates from a linked file.

Option Revit Category Notes
Walls Walls (Basic, Stacked, Curtain) Supports both structural and architectural walls.
Floors Floors Includes structural and architectural floor types.
Ceilings Ceilings Compound ceilings supported; in-place ceilings may have limitations.
Beams / Columns Structural Framing & Columns Openings are cut as voids; requires compatible family geometry.
⚠
Warning Enabling Beams / Columns may affect structural calculations. Always consult the structural engineer before creating openings in structural framing members.

3. Opening Settings

Configure how openings are created after clashes are detected.

Opening Type

OptionDescriptionWhen to Use
Use Native Openings Creates Revit built-in wall/floor openings using the Opening by Face or Shaft Opening tools internally. Standard projects where no sleeve family is required; simpler documentation.
Use Family Sleeves Places a user-specified sleeve family at each clash location, sized to match the MEP element plus the defined clearance. Projects requiring fire-stopping sleeves, cast-in inserts, or mechanical sleeves per specification.

Sleeve Family

Available only when Use Family Sleeves is selected. Click the field to browse and select a loaded sleeve family from the project. The family must be a face-based or wall/floor-based generic model with a Nominal Diameter or Width × Height parameter for automatic sizing.

Added Clearance (mm)

Specifies the additional gap in millimetres added around the MEP element's bounding profile when calculating the opening size.

✅
Tip A clearance of 50 mm (default) satisfies most fire-stopping and insulation wrap requirements. Verify against your project specification and local building code before changing this value.

Merge Overlapping Clashes

When enabled (recommended), adjacent clashes within the same host element whose bounding boxes overlap — accounting for the added clearance — are merged into a single larger opening. This prevents closely-spaced openings that would undermine structural integrity and produce cluttered documentation.

Only in Active View

Limits the scan scope to elements visible in the currently active 3D or plan view. Useful for targeted clash resolution in a specific area or zone without processing the entire model.

ℹ
Note Only in Active View respects Revit's view-specific visibility/graphics overrides and Section Box constraints. Elements hidden by a filter or temporarily hidden in the active view will be excluded from the scan.

Results Table

After running Scan For Clashes, the results table on the right side of the dialog lists every detected clash with the following columns:

Column Description
✓ Row-level checkbox. Uncheck to exclude specific clashes from the Create Openings operation.
MEP Type Category of the intersecting MEP element (Pipe, Duct, or Cable Tray).
MEP Source Model source: Host Model or the filename of the linked Revit model.
Host Type Category of the host element being cut (Wall, Floor, Ceiling, Beam/Column).
Host ID Revit Element ID of the host element. Click to zoom and select the element in the model.
Status Current opening state:
● Pending — clash detected, opening not yet created.
● Done — opening successfully placed.
● Conflict — opening could not be placed (see Troubleshooting).

Scan Summary

The Scan Summary panel at the bottom of the left pane provides a quick totals report after each scan or opening creation run:

  • Clashes found — total number of MEP-to-host intersections detected.
  • Openings done — total number of openings successfully placed in the current session.

  1. Configure selections
    Check the required MEP categories (Pipes, Ducts, Cable Trays) and host types (Walls, Floors, etc.). Set the clearance value and choose between Native Openings or Family Sleeves.
  2. Click "Scan For Clashes"
    The tool analyses all intersections between selected MEP and host elements across the host model and linked files. Results populate the table on the right.
  3. Review results
    Inspect each row. Click a Host ID to navigate to the clash location in the model. Uncheck any rows you wish to exclude.
  4. Click "Create Openings"
    The tool places native openings or sleeve families for all checked rows. A Revit transaction is created — use Ctrl+Z to undo the entire batch if needed.
  5. Verify and document
    Check the Status column for any Conflict items. Review the Scan Summary. Save the model and coordinate with the structural discipline.
🚫
Important Always run Scan For Clashes again after any MEP routing changes or model reloads to ensure the results table reflects the current model state before creating additional openings.

Troubleshooting

No clashes found after scan

  • Confirm that at least one MEP category and one host type are checked.
  • Verify that MEP elements and host elements overlap geometrically in 3D. Use a 3D view with a Section Box to inspect intersections.
  • If Only in Active View is checked, ensure the active view's Section Box includes the expected area.
  • Confirm linked models are loaded (status bar reports linked model count).

Status shows "Conflict" after Create Openings

  • The host element may be a non-cuttable family (e.g., some curtain wall panels). Convert to a cuttable host or use a different opening method.
  • The opening size exceeds the host element's geometric bounds. Reduce the clearance value or split the MEP run.
  • Permissions: you may not have edit access to the workset owning the host element in a workshared model. Request ownership and retry.

Sleeve family not appearing in the dropdown

  • Load the sleeve family into the project via Insert → Load Family before opening the dialog.
  • Ensure the family category is Generic Model, Mechanical Equipment, or a supported pipe/duct-related category.
